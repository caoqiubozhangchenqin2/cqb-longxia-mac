## P1｜路由边界说明（建议统一口径）

> 目标：避免“串群响应”、减少误触发。

### 全局建议
- 各业务 Agent：仅在各自绑定群内响应。
- `main`：作为默认兜底，仅处理未匹配路由的消息；不做群内业务执行。
- 涉及外发/群发/公开发布：先确认后执行。

### Agent 边界（当前建议版）
- **recruiter（招聘专员）**
  - 只处理 Agent 创建、身份配置、绑定编排相关任务。
  - 不直接代表业务 Agent 对外发布。
- **team-manager**
  - 仅处理球队管理相关话题。
- **fund-assistant（基金助理）**
  - 仅处理基金观察、记录、分析辅助。
  - 不做未经确认的投资建议外发。
- **course-advisor（课程顾问）**
  - 仅处理课程咨询、排课、学员沟通草案。
- **height-recorder（身高记录师）**
  - 仅处理身高记录与追踪。
- **main（默认）**
  - 默认兜底，不承接具体业务角色；建议后续绑定专用“总控群”或缩小默认范围。

## P2｜新群接入 Checklist（创建/绑定/验收一次过）

### A. 创建阶段
- [ ] 明确 Agent 名称（id + 显示名 + emoji）
- [ ] 明确 1 句话职责（做什么）
- [ ] 明确边界（不做什么）
- [ ] 明确目标群（群名称 + chat_id）

### B. 配置阶段
- [ ] 创建 Agent 工作目录
- [ ] 设置 identity（语气、人设、语言）
- [ ] 选择模型档位（成本优先 / 质量优先）
- [ ] 挂载必要技能（如 Feishu 文档、表格、天气等）

### C. 绑定阶段
- [ ] 绑定到指定飞书群
- [ ] 校验无重复冲突路由
- [ ] 校验默认兜底是否合理（main 是否会误接）

### D. 验收阶段
- [ ] 群内触发词测试（至少 2 条）
- [ ] 异常话题拒答测试（越权场景）
- [ ] 输出“Agent 卡片”存档（ID/目录/模型/群ID/边界）
- [ ] 记录负责人与变更日期
